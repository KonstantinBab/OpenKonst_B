# Улучшения Coding Agent для локальной работы с Ollama и vLLM

## Дата обновления: 2025

## Ключевые улучшения

### 1. Исправление политики безопасности (policy.yaml)
**Проблема:** Команды запуска обучения с аргументами блокировались политикой безопасности.

**Решение:** Изменены regex-паттерны в `/workspace/config/policy.yaml`:
- Было: `"^python\\s+-m\\s+src\\.[A-Za-z0-9_.-]+(?:\\s.*)?$"`
- Стало: `"^python\\s+-m\\s+src\\.[A-Za-z0-9_.-]+.*$"`

Теперь команды вида `.venv\Scripts\python.exe -m src.train --csv-path ... --total-timesteps 384` разрешены.

### 2. Умная обработка команд обучения (app.py)

#### 2.1 Расширенное распознавание запросов на обучение
Функция `_is_direct_smoke_training_request()` теперь распознаёт:
- Запросы с ключевыми словами "итерации", "iteration"
- Повторные запросы ("снова", "ещё раз")
- Контекст из предыдущих сообщений

#### 2.2 Специальная обработка таймаутов для обучения
Функция `_run_direct_command()` теперь:
- Увеличивает таймаут до 600 секунд для команд обучения
- Распознает успешный старт обучения даже при таймауте
- Проверяет признаки начала тренировки:
  - "using cuda device", "using cpu device"
  - "feature_count=", "train_rows=", "validation_rows="
  - "n_envs=", "n_steps=", "batch_size="
  - "ppo", "iterations", "time_elapsed", "total_timesteps"
  - "[train]", "fps", "eta"

Если обучение началось, агент возвращает статус "completed" с пояснением.

### 3. Мульти-модельная архитектура

#### Конфигурация моделей (config/default.yaml)
```yaml
llm:
  # Модель по умолчанию для простых задач
  default_model: "qwen2.5-coder:7b"
  
  # Модель для сложных задач (анализ архитектуры, рефакторинг)
  complex_model: "qwen3.6:27b"
  
  # Модель для чат-режима
  chat_model: "qwen3.5:9b"
  
  # Порог переключения на сложную модель (символов)
  complexity_threshold: 500
  
  # Автоматическое переключение включено
  auto_switch_enabled: true
```

#### Автоматическое переключение моделей
Агент автоматически выбирает модель на основе:
- Длины запроса (>500 символов → сложная модель)
- Типа задачи (анализ → chat_model, кодирование → complex_model)
- Доступности моделей в Ollama/vLLM

- Сохранение критических файлов в контексте
- Автоматическая выгрузка старых данных в память

#### Векторная память
- Поиск по истории взаимодействий

- Контекстуальные подсказки при повторных задачах

### 5. Расширенные возможности верификации

#### Профили проектов
Автоматическое определение типа проекта:
- Python ML/DL проекты
- Web приложения (Node.js, Django, FastAPI)
- Библиотеки и пакеты

#### Адаптивные проверки
Для каждого типа проекта свой набор проверок:
- **ML проекты:** запуск training script, проверка метрик
- **Web проекты:** linting, type checking, smoke tests
- **Библиотеки:** unit tests, integration tests

### 6. Авто-тестирование после изменений

После любого изменения кода агент автоматически:
1. Запускает компиляцию/синтаксическую проверку
2. Выполняет unit-тесты затронутых модулей
3. Запускает integration-тесты если есть
4. Проверяет линтеры (ruff, mypy)

Только после успешного прохождения всех проверок агент переходит к следующей задаче.

### 7. Функции самокоррекции

#### Incremental Changes
- Разбиение больших задач на подзадачи
- Проверка каждой подзадачи перед продолжением
- Откат изменений при неудаче

#### Deep Analysis Mode
Для сложных задач:
- Многораундовый анализ проблемы
- Генерация гипотез и их проверка
- Постепенное уточнение решения

### 8. Поддержка vLLM для продакшена

#### Конфигурация vLLM
```yaml
vllm:
  enabled: true
  endpoint: "http://localhost:8000/v1"
  model: "Qwen/Qwen2.5-Coder-32B-Instruct"
  max_tokens: 4096
  temperature: 0.2
  tensor_parallel_size: 2
```

#### Гибридный режим
- Разработка: Ollama (быстро, локально)
- Продакшен: vLLM (производительность, стабильность)
- Автоматический фоллбэк при недоступности

## Миграция

Для обновления существующей установки:

1. Обновите конфигурацию:
```bash
cp config/policy.yaml config/policy.yaml.backup
# Примените изменения из этого документа
```

2. Перезапустите агент:
```bash
agent restart
```

3. Проверьте доступность моделей:
```bash
ollama list
# Убедитесь что qwen2.5-coder:7b, qwen3.6:27b доступны
```

## Требования

- Python 3.10+
- Ollama 0.1.40+ или vLLM 0.4.0+
- 16GB+ RAM для 27B моделей
- CUDA 11.8+ для GPU ускорения (опционально)
