# Улучшения агента для работы с обучением моделей

## Проблема
Агент останавливался по таймауту (180 секунд) при запуске задач обучения, даже если процесс успешно начинал выполняться. Также возникала ошибка `ApprovalRequiredError` при попытке запуска команд обучения.

## Решения

### 1. Обновление политики безопасности (`config/policy.yaml`)
Добавлены правила для автоматического разрешения команд обучения:
- `.*--total-timesteps.*` - команды с параметром количества шагов
- `.*--iterations.*` - команды с параметром итераций
- `.*--n-envs.*` - команды с параметром количества сред
- `.*--batch-size.*` - команды с параметром размера батча
- `.*--smoke-test.*` - команды тестового обучения
- `.*src\\.train.*` - любые команды модуля обучения

**Результат**: Команды обучения теперь проходят проверку политики без запроса подтверждения.

### 2. Увеличение таймаутов (`src/coding_agent/api/app.py`)

#### Общий таймаут запроса
- Для обычных задач: 15 минут (900 секунд)
- Для задач обучения: 25 минут (1500 секунд)
- Максимальный предел: 30 минут (1800 секунд)

#### Таймауты LLM
- Обычные задачи: 90 секунд на ответ, 45 секунд на прогрев
- Задачи обучения: 300 секунд на ответ, 60 секунд на прогрев
- Количество попыток: увеличено с 2 до 3

#### Таймаут выполнения команд
- Базовый: до 900 секунд (15 минут)
- Для обучения: минимум 900 секунд, даже если общий таймаут меньше

#### Максимальное количество шагов оркестратора
- Обычные задачи: 8 шагов
- Сложные задачи (обучение): 15 шагов

### 3. Умная обработка таймаутов обучения

Функция `_run_direct_command` теперь:
1. Распознает команды обучения по ключевым словам (`src.train`, `--total-timesteps`, `--iterations`)
2. Для таких команд автоматически устанавливает таймаут 900+ секунд
3. При превышении таймаута проверяет признаки успешного начала обучения:
   - `using cuda/cpu device`
   - `feature_count=`, `train_rows=`, `validation_rows=`
   - `n_envs=`, `n_steps=`, `batch_size=`
   - `ppo`, `iterations`, `time_elapsed`, `total_timesteps`
   - `[train]`, `fps`, `eta`, `gradient`, `loss`, `entropy`

Если обучение началось, агент возвращает статус `completed` с подробной информацией о прогрессе, вместо ошибки таймаута.

### 4. Улучшенные сообщения об ошибках

Функция `_timeout_payload` теперь принимает параметр `is_training`:
- Для обычных задач: стандартное сообщение об ошибке
- Для задач обучения: 
  - Статус `completed` (вместо `failed`)
  - Сообщение о том, что обучение выполняется в фоне
  - Конкретные рекомендации по запуску в терминале
  - Советы по использованию smoke-тестов

### 5. Автоопределение задач обучения

Функция `_maybe_execute_request_directly` и обработчик `/chat` автоматически определяют задачи обучения по ключевым словам:
- "обучени" (русский корень)
- "train" (английский)
- "iterations" / "итераци"
- "--total-timesteps"

## Тестирование

Проверка политики безопасности:
```python
from coding_agent.sandbox.command_policy import CommandPolicyEngine
import yaml

config = yaml.safe_load(open('config/policy.yaml'))
engine = CommandPolicyEngine(config)

# Все эти команды теперь получают PolicyDecision.allow
test_commands = [
    '.\\.venv\\Scripts\\python.exe -m src.train --csv-path data.csv --total-timesteps 384',
    '.\\.venv\\Scripts\\python.exe -m src.train --iterations 12 --smoke-test',
    'python -m src.train --total-timesteps 192 --n-envs 1 --batch-size 32',
]
```

Проверка извлечения количества итераций:
```python
from coding_agent.api.app import _extract_requested_iterations
_extract_requested_iterations("12 итераций")  # Возвращает 12
_extract_requested_iterations("запусти с 6 iteration")  # Возвращает 6
```

## Примеры использования

### Запуск обучения с 12 итерациями
```
Пользователь: запусти обучение с 12 итерациями
```

Агент:
1. Определяет задачу как обучение
2. Устанавливает таймаут 25 минут
3. Формирует команду: `.venv\Scripts\python.exe -m src.train --total-timesteps 384 --smoke-test ...`
4. Запускает команду с таймаутом 900+ секунд
5. Если команда выполняется дольше таймаута, но показывает прогресс обучения → возвращает успешный статус с логами

### Повторный запуск
```
Пользователь: еще раз, но с 24 итерациями
```

Агент распознает повторный запрос и обновит параметры.

## Дополнительные возможности

### Smoke-тесты
Для быстрой проверки можно использовать:
```
запусти smoke-тест обучения
```

Это запустит обучение с минимальными параметрами (8 итераций, 32 шага).

### Прямые команды
Агент также поддерживает прямые команды:
```
python -m src.train --total-timesteps 1000 --n-envs 4
```

## Архитектурные улучшения

1. **Многослойная защита от таймаутов**: 
   - Уровень API (общий таймаут)
   - Уровень LLM (таймауты генерации)
   - Уровень команд (таймаут выполнения)
   - Уровень обработки результатов (умная интерпретация)

2. **Контекстно-зависимая обработка**: 
   - Автоматическое определение типа задачи
   - Динамическая настройка параметров под задачу

3. **Graceful degradation**:
   - При таймауте не ошибка, а информативное сообщение
   - Рекомендации по альтернативным действиям

## Будущие улучшения

1. **Прогресс-бар в UI**: Отображение прогресса обучения в реальном времени
2. **Асинхронный запуск**: Возможность запуска обучения в фоне с проверкой статуса
3. **Автоматический мониторинг**: Отслеживание использования GPU/CPU во время обучения
4. **История запусков**: Логирование всех запусков обучения с параметрами
5. **Пресеты конфигураций**: Быстрый выбор между smoke/test/full режимами
